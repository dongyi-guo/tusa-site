# Understanding Club Identity: The Keystone Account Model

[][1]

This guide explains how clubs are set up in the TUSA system and why it works the way it does. Reading this once will save you a lot of confusion later — especially during handovers and reaffiliation.

------------------------------------------------------------------------

## The Club President Account: A Permanent Position[][5]

This is the most important concept to understand.

### The “Throne” Model[][6]

Think of the club president account like a throne in a monarchy:

| Element           | Description                                                  |
| ----------------- | ------------------------------------------------------------ |
| **The Throne**    | The user account itself — permanent, never changes           |
| **The Monarch**   | The person sitting on it — changes over time                 |
| **The Nameplate** | The personal details on the profile — gets updated when the person changes |

**The throne (account) stays. The person sitting on it changes.**

### Why This Matters: The Store Connection[][7]

The club president account is **permanently linked** to your club’s store as the owner. This connection:

Cannot be transferred to a different account

Cannot be deleted without losing the store

Must persist year after year

**This is why we don’t delete the president account when leadership changes.** Deleting it would orphan the entire store.

### What Stays the Same vs What Changes[][8]

| Permanent (The Account)                      | Updates Each Year (The Person) |
| -------------------------------------------- | ------------------------------ |
| Username and login email                     | Real name                      |
| Public role label (“President — Chess Club”) | Personal email                 |
| Store ownership                              | Phone number                   |
| Group admin status                           | Student ID                     |
| Faculty / course                             |                                |
| Address                                      |                                |
| Club role title                              |                                |

### The Public Role Label[][9]

The account’s public-facing label is set to the **role**, not the person:

    Display Name: "President — Chess Club"
    First Name: "President"
    Last Name: "Chess Club"

This is deliberate. The account represents the **position**, not the individual. So when a new person takes over, the public label is already correct.

### Where the Real Person’s Details Live[][10]

The actual person’s information is stored in your **profile fields** — these are the ones you fill in during reaffiliation:

**Real Name** — e.g. “Sarah Smith”

**Personal Email** — e.g. “sarah.smith@utas.edu.au”

**Phone** — e.g. “0412 345 678”

**Student ID** — e.g. “123456”

**This is what the rolodex and contact systems use** — so they always show the current person, not just the position title.

## The Club Admin Rolodex[][11]

The rolodex is an internal directory that shows TUSA staff and club administrators the **real people** behind each club role.

### Why the Rolodex Exists[][12]

| Scenario                  | What’s Needed                                                |
| ------------------------- | ------------------------------------------------------------ |
| **Emergency at an event** | Need to call the actual president, not “President — Chess Club” |
| **Compliance follow-up**  | Need to email the real person about missing paperwork        |
| **Year-round contact**    | Need to know who’s currently in each role                    |
| **Handover verification** | Need to confirm new committee details are correct            |

### What the Rolodex Shows[][13]

The rolodex pulls from your profile fields, so it always reflects the **current person** in each role:

| Field          | Example                   |
| -------------- | ------------------------- |
| Real Name      | “Sarah Smith”             |
| Personal Email | “sarah.smith@utas.edu.au” |
| Phone          | “0412 345 678”            |
| Club Role      | “President”               |
| Student Status | “Yes”                     |
| Student ID     | “123456”                  |

### When the Rolodex Updates[][14]

The rolodex updates automatically because it pulls from live profile data:

| Event                        | What Happens                         |
| ---------------------------- | ------------------------------------ |
| **Initial Affiliation**      | New club’s officers added to rolodex |
| **Annual Reaffiliation**     | All officer details refreshed        |
| **Mid-year Profile Updates** | Reflect immediately                  |

### Who Can See the Rolodex[][15]

The rolodex is an **internal tool** — not visible to regular members or the public. It’s accessible to:

TUSA Staff

Club administrators (for their own club)

This protects personal contact details while still allowing the right people to reach the right person.

## Handover Process: Updating the Throne[][16]

When a new president takes over:

### What Actually Happens[][17]

**The account stays** — same login, same store link, same group admin status

**Login credentials transfer** — the new president gets the password from the outgoing one

**Profile fields update** — the new person’s details replace the old ones

**Personal email must be unique** — so notifications reach the right human

### During Reaffiliation[][18]

When you complete reaffiliation:

You confirm **you** are the current person in the president seat

Your personal details (name, email, phone, student ID) are updated

The underlying account stays intact

Your store and group connections are preserved

### The Personal Email Field is Critical[][19]

The **login email** stays the same year after year (often a club email like `president@chessclub.org`). But your **personal email** field must always be the **current person’s individual email**.

| Field          | Purpose                               | Example                 |
| -------------- | ------------------------------------- | ----------------------- |
| Login Email    | Account access — usually a club email | president@chessclub.org |
| Personal Email | Reaches the actual human              | sarah.smith@utas.edu.au |

**Why does this matter?** When TUSA needs to contact the president personally, they reach Sarah, not “whoever happens to check the club inbox.”

## Committee Members: A Different Model[][20]

Committee members work differently from the president.

### Two Scenarios[][21]

**Scenario A: Role-Based Club Emails**

e.g. `treasurer@club.org`, `secretary@club.org`

Each role is a permanent account (like mini-thrones)

Person changes, account stays

Profile fields updated during reaffiliation

**Scenario B: Personal Accounts**

e.g. `sarah.smith@utas.edu.au` acting as treasurer

Account belongs to the person, not the role

When they leave the committee, they’re demoted (not deleted)

The next person uses (or creates) their own account

### Committee Member Access[][22]

| What They Get        | What It Means                          |
| -------------------- | -------------------------------------- |
| Group admin status   | Can manage group settings and posts    |
| Store staff access   | Can manage products and orders         |
| Committee role badge | Identifies their role in the directory |

When someone is removed from the committee:

**If they’re a club member:** demoted to regular member (still in the group)

**If they’re not a member:** removed from the group entirely

## Practical Implications[][23]

### For Presidents[][24]

**You inherit an account** — you don’t create a new one

**Update your details** — make sure your profile has YOUR info, not the previous president’s

**Your personal email matters** — use YOUR email, not the club one

**The store is yours to manage** — but you’re a steward, not the owner

### For Committee Members[][25]

**Your access is tied to your role** — leave the committee, lose admin access

**Your personal account stays yours** — unless you’re using a role-based email

**Update your details annually** — during reaffiliation

### For Understanding Your Club[][26]

Your club is the combination of:

A permanent **president account** (the keystone)

A **public store**

A **members-only group**

**Committee accounts** (supporting structure)

**Regular member accounts** (the membership)

The president account is the **keystone** that holds the store and group together. Remove it, and the arch collapses.

## Separating Personal from Club Activity[][27]

The system distinguishes between personal and club activity.

### Personal Activities (Your Own Account)[][28]

If you have a personal account separate from any club role:

Personal purchases (events, merchandise from other clubs)

Personal memberships to other clubs

Personal profile and connections

### Club Role Activities (Club Admin Account)[][29]

When you’re acting as club president or committee:

Managing the store

Posting to the club group

Handling club admin tasks

Represented by the club role label, not your personal name

### The Gear Hire Example[][30]

**Club purchase:** Hiring equipment for a club event — use the club admin account\
**Personal purchase:** Buying something for yourself — use your personal account

Keep these separate to maintain clean records and the right access on each side.
